# /brain:unpin

Remove a memory from the [always-present tier](/docs/concepts/pinned-memory/). It will no longer load automatically at session start and returns to normal recall and decay.

## Usage

```
/brain:unpin [id|query]
```

**Arguments:**
- `[id|query]` — A memory ID (`mem_…`) to unpin directly, or a query. With a query, the agent inspects the current pins (`~/.brain/pinned.json`) or recalls candidates and confirms which pin to remove.

## What It Does

1. **Resolves the target** — by ID, or by matching against the current set of pins.
2. **Unpins it** — removes the memory from `~/.brain/pinned.json`.
3. **Confirms:**

```
📌 Unpinned "Deploy via Cloudflare Workers" — no longer auto-loaded.
```

```bash
brain unpin <id>
```

<Callout type="info">
  Unpinning returns the memory to normal [recall](/docs/commands/remember/) and [decay](/docs/concepts/strength-decay/). An independent `stable` flag, if it was set, is left untouched — so a memory can stop loading every session while still never fading.
</Callout>

## Example

```
User: /brain:unpin cloudflare deploy

Agent: Currently pinned and matching your query:
  • Decision: "Deploy via Cloudflare Workers, not Vercel" (mem_20260512_c4d8e2)

📌 Unpinned "Deploy via Cloudflare Workers, not Vercel" — no longer auto-loaded.
   It's back in normal recall and will decay over time like any other memory.
```
